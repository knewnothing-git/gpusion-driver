01 — Problem Statement
300 million laptops.
Zero GPUs.

India's middle class owns budget laptops in the ₹30,000–₹60,000 range. Every serious local AI workload requires hardware they cannot afford — or configuration complexity they cannot navigate.

The Access Gap

Running Stable Diffusion requires a minimum ₹22,000 GPU. Running Llama 3 locally needs 8GB VRAM — another ₹25,000+. Cloud APIs cost ₹800–2,000/month. None of these are viable for a student in Patna or a freelancer in Coimbatore.

03 — Technical Architecture
The three-layer illusion.

Applications call standard GPU APIs. GPUsion intercepts at the driver layer and routes to an optimized inference engine. Every app believes a GPU is present. None of them need to change a single line of code.

Application Layer — untouched
Stable Diffusion · Ollama · Whisper · LM Studio · Any DirectML / Vulkan / OpenCL app
▼ standard GPU API calls (DirectML / Vulkan / OpenCL)
GPUsion Virtual Driver — the illusion layer
WDDM Kernel Driver · API Interceptor · GPU Command Translator · Fake Device Emulator · VRAM Proxy
▼ routed inference workloads
Inference Engine — Phase 1: CPU · Phase 2: FPGA · Phase 3: ASIC
llama.cpp · ONNX Runtime · OpenBLAS · AVX2/AVX-512 SIMD · INT4/INT8 Quantization
▼ hardware compute
Hardware Layer
Phase 1: CPU RAM as VRAM proxy · Phase 2: FPGA systolic array via USB · Phase 3: Custom ASIC

04 — Open Source Strategy
Open source is the weapon, not the product.

The driver and inference layer are MIT-licensed. This creates trust, distribution, and acquisition interest simultaneously. The hardware dongle is the commercial layer — open source makes it inevitable.

GitHub Repositories — MIT Licensed
  • gpusion-driver — WDDM virtual adapter. The crown jewel.
  • gpusion-inference — CPU translation layer
  • gpusion-firmware — FPGA/ASIC firmware (Phase 2)
  • gpusion-installer — One-click Windows installer
  • gpusion-benchmark — Standardized benchmarks

05 — Roadmap
From illusion to silicon.

Eight milestones across 24 months. Each phase validates before the next begins. No speculative spending, no phantom features.

M 1–2
Driver Skeleton
WDDM virtual adapter registers in Device Manager. Basic DirectML passthrough to CPU ONNX runtime.
M 2–3
Ollama Integration
GPUsion virtual GPU auto-selected by Ollama. First LLM runs end-to-end on a no-GPU laptop.
M 3–4
Installer + Non-technical UX
One-click installer. Zero command line. First test with a non-technical user — if they fail, we failed.
M 4–5
GitHub Public Launch
HackerNews + Reddit launch. Target: 500 GitHub stars within 30 days. First press coverage.
M 5–6
Benchmarks + Press
Publish standardized performance data vs. real GPU baselines. Outreach to Indian tech media.
M 6–9
FPGA Hardware POC
First physical dongle. FPGA-based inference. Demonstrate 2× performance vs. software-only.
M 9–12
Hardware v1 — First Sales
Manufacturable GPUsion Stick. First 500 units. Flipkart + Amazon India listing. Target NPS > 40.
M 12–24
ASIC Design + Scale
Custom chip design. TSMC 28nm tape-out funding. First OEM MOU. Proudly Made in India.
Key Risks
WDDM Driver Signing
Microsoft requires EV code signing certificate (~$300/yr) for kernel drivers.
Budget from Month 1. Test signing mode during development.
Performance Gap
CPU inference is 5–10× slower than a real GPU in most workloads.
Aggressive INT4 quantization, model selection guidance, honest benchmarks. Phase 2 closes the gap.
CUDA Apps
Applications hardcoding CUDA calls bypass DirectML entirely.
Clearly scoped: Phase 1 targets DirectML/ONNX apps only. CUDA is a Phase 3 stretch goal.
Large Player Clone
Microsoft or Intel could build a similar abstraction layer.
Speed + community moat. First-mover in India. Open source is hard to kill — forks survive acquisitions.
